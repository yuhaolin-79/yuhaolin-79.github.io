1.3 string的使用

1.3.1 string的简介

  1. 定位：string是C++标准库的重要组件，用于字符串处理。
  2. 使用前提：需在代码中包含头文件#include<string>。
  3. 核心特点：与传统的char[]相比，string是高度封装的类型，支持便捷的字符串操作（如拼接、截取、匹配等）。

string的核心特性

  1. 字符串管理：
    封装了字符串的存储与管理，自动处理内存分配/释放，无需手动管理内存。
  2. 动态大小调整：
    可根据需求自动调整字符串容量，添加/删除字符时会自动扩容，保证存储空间充足。
  3. 安全性：
    提供越界访问检查，避免访问超出字符串范围的字符，降低错误风险。
  4. 迭代器支持：
    支持迭代器，可通过迭代器遍历字符串中的字符，实现字符级别的操作。
  5. 兼容性：
    属于C++标准库组件，在C++中广泛应用，与其他标准库组件、C++语言特性兼容。

1.3.3 基本操作

c_str()的作用：
std::string的成员函数c_str()会返回一个以空字符结尾的C风格字符串指针（const char*类型），用于兼容C语言的函数（如printf、scanf等）。

std::string常用操作（代码+解析）

1. 获取字符串长度（length/size）

  • 功能：length()与size()作用完全一致，均返回字符串的字符个数（上述示例输出为13）。

2. 拼接字符串（+/append）

  • 功能：两种方式均实现字符串拼接，示例输出均为Hello, World!；
  • 区别：+生成新字符串，append直接修改原字符串（示例中str1会被修改为Hello, World!）。

3. 字符串查找（find）

  • 功能：查找子串首次出现的起始索引，找到则返回位置（示例中pos=7），未找到则返回std::string::npos。

4. 字符串替换（replace）

  • 参数说明：replace(起始索引, 替换长度, 新子串)；
  • 示例输出：将索引7开始的5个字符（World）替换为Universe，最终输出Hello, Universe!

5. 提取子字符串（substr）

  • 功能：substr(起始索引, 长度)从字符串中截取指定长度的子串。
  • 示例结果：从索引7开始截取5个字符，输出为World。

    注意：不要越界

6. 字符串比较（compare）

  • 返回值规则：
    • result == 0：两个字符串相等；
    • result < 0：str1字典序小于str2；
    • result > 0：str1字典序大于str2；
  • 示例结果：Hello字典序小于World，输出String 1 is less than String 2.。

7. string的字典序比较规则

string重载了不等号（如<、>），直接用运算符比较时，遵循字典序规则：

  • 从第一个字符开始逐位比较，遇到第一个不相等的字符时，按该字符的ASCII值大小确定整体顺序；
  • 若前面字符都相等，短字符串更小。

示例：

  • aaaa < bbbb（首字符a的ASCII小于b）；
  • azz < baaa（首字符a < b，后续字符无需比较）；
  • azzzzzzzzz < b（同理，首字符a < b）；
  • lanqiao == lanqiao（所有字符完全匹配）。

推荐使用不等号直接比较

8.string的两种遍历方法（代码+解析）

1. 循环枚举下标
  • 功能：通过[i]访问字符串的第i个字符（下标从0开始），支持读写操作；
  • 输出：Hello。
2. auto枚举（范围for循环）

auto遍历有两种形式，核心区别是“是否取引用”：

形式1：auto i : s（拷贝遍历）
  • 特点：i是字符串中每个字符的拷贝，修改i不会影响原字符串；
  • 输出：Hello。
形式2：auto &i : s（引用遍历）
  • 特点：&表示取引用，i直接关联原字符串的字符，修改i会同步修改原字符串；
  • 输出：遍历输出Hello，最终输出s的结果为aaaaa。

总结

  • 下标遍历是最基础的方式，适合需要明确下标位置的场景；
  • auto遍历更简洁，若需修改原字符串，必须加&取引用；若仅读取，可省略&（或加const auto &提升效率）。
